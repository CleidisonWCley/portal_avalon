# RELATÓRIO — Portal Avalon V6.0
## Módulo Raid e Estratégias de Combate

## Resumo
A versão V6.0 adiciona ao Portal Avalon a nova aba **Raid**, voltada para consulta estratégica de bosses, elementos e composições recomendadas com base na **GTales.top API**.

O objetivo foi criar um novo bloco funcional sem alterar áreas já estabilizadas do projeto, preservando Liga, Hall, Registro, Galeria, OCR, mapas e canvas de cards.

## Menu atualizado
A aba **Raid** foi adicionada entre **Registro** e **Galeria**.

```text
Salão | Hall | Buscar | Registro | Raid | Galeria | Liga
```

Motivo: Registro e Raid são áreas diretamente ligadas à atividade principal da guilda. Registro mostra desempenho; Raid mostra estratégia.

---

## Página Raid criada
A página `raid.html` inclui:

- cabeçalho com mascotes Cley Raid e Olimpio Raid;
- título **Estratégias de Raid**;
- identificação da fonte: **GTales.top API**;
- aviso de fonte externa;
- filtros de boss e elemento;
- botão buscar times;
- botão recarregar dados;
- botão limpar consulta;
- painel de resultados;
- estados de loading, erro e sem resultados.

---

## Integração API
Implementado em `web/assets/js/raid.js`:

```text
GET https://gtales.top/api/raids
GET https://gtales.top/api/raids?boss={boss}&element={element}
```

A primeira chamada carrega a lista de bosses e elementos disponíveis. A segunda carrega os times recomendados para a combinação selecionada.

## Imagens externas
A página tenta carregar imagens externas da GTales.top:

```text
/assets/bosses/{boss}.webp
/assets/heroes/{heroesAtr}.webp
/assets/weapons/{weaponsAtr}.webp
/assets/access/{access}.webp
```

Caso a imagem falhe, o card mostra um fallback textual sem quebrar o layout.

---

## Cache local
A V6.0 usa `localStorage` para cache simples:

```text
portal_avalon_raid_api_list_v60
portal_avalon_raid_api_query_v60_{boss}_{element}
portal_avalon_raid_last_search_v60
```

O cache reduz chamadas repetidas e melhora a experiência se a API estiver instável temporariamente.

---

## Segurança do conteúdo externo
O campo `infos`, que pode vir como HTML da API, é convertido para texto limpo antes da renderização. A V6.0 evita inserir HTML externo diretamente na página.

## Observação sobre hospedagem
A V6.0 preserva o modelo de site estático e pode ser hospedada em GitHub Pages ou Cloudflare Pages. Caso a API bloqueie requisições diretas por CORS no navegador, a solução futura recomendada é um proxy simples com Cloudflare Worker/Function, sem abandonar a hospedagem gratuita.
