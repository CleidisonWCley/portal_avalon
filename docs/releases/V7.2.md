# Relatório de Atualização — Portal Avalon V7.2

## Hall com Histórico Rotativo de Raids

## Objetivo

Blindar o Hall da Evolução contra variações causadas por boss, elemento e baixa frequência, adotando uma média histórica recente no lugar da comparação exclusiva com uma única raid anterior.

## Dados utilizados

- Fonte oficial atual: `web/data/raids/raid_atual.json`, gerada pelo OCR.
- Semente histórica: `Raid danos.xlsx`.
- Colunas históricas: `Dano 3`, `Dano 2` e `Dano 1`.
- `Dano 4` foi usado apenas para conferência; em caso de divergência, o OCR venceu.

## Roster filtrado

Foram mantidos somente os 28 nomes presentes na raid atual.

Ignorados por não fazerem parte do roster atual:


## Blindagens aplicadas

- Raid histórica com frequência desconhecida entra na média e marca **Base estimada**.
- Raid histórica com frequência conhecida abaixo de `15/21` é excluída da média.
- `Wagnero`, na raid `Dano 3`, foi registrado com `6/21` e não influencia sua média.
- É exigido mínimo de duas raids anteriores válidas.
- Presença atual mínima para o Hall: `15/21`.
- A evolução é calculada por:

```text
média base = média das raids anteriores válidas
evolução absoluta = dano atual - média base
percentual evolutivo = evolução absoluta / média base × 100
```

## Atualizações de interface

- Ordenação por percentual evolutivo.
- Desempate por evolução absoluta e dano atual.
- Exibição de média base, percentual e confiança histórica.

- Dano anterior direto.
- Média base.
- Evolução absoluta.
- Percentual evolutivo.
- Quantidade de raids válidas.
- Base oficial, estimada, parcial ou insuficiente.

### Registro

- Novas colunas para média base, percentual evolutivo e confiança histórica.
- Filtro “Sem comparativo” renomeado visualmente para “Base insuficiente”.

### Canvas da ficha

- Mantém a imagem de patente em alta resolução.
- Passa a destacar dano atual, média base, evolução percentual e frequência.
