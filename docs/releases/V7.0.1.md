# Relatório de Atualização — Portal Avalon V7.0.1

## Refinamento Visual da Aba Raid

## 1. Objetivo

A versão **Portal Avalon V7.0.1** refina a experiência visual da aba **Raid** após a integração oficial com o Worker Cloudflare realizada na V7.0.

O foco foi manter a integração funcionando e melhorar a apresentação para uso real pelos membros da guilda, com uma área de consulta mais limpa, imagens de heróis mais estáveis e bloco de chains mais legível.

---

## 2. Arquivos alterados

```text
web/pages/raid.html
web/assets/js/raid.js
web/assets/css/styles.css
READM_AVALON_VF_1.md
RELATORIO_V7_0_1_ATUALIZACAO.md
```

---

## 3. Área de consulta

A área de consulta foi simplificada para reduzir a aparência técnica da página.

Alterações realizadas:

```text
- Removido o aviso grande sobre API externa.
- Adicionada referência discreta à fonte Guardian Tales TOP.
- Boss e elemento foram mantidos na primeira linha.
- Botões Buscar times, Recarregar dados e Limpar consulta foram organizados em linha horizontal abaixo dos selects.
- A mensagem de status ficou mais discreta.
```

---

## 4. Imagens dos heróis

Foi aplicado um refinamento de estabilidade nos avatars dos heróis.

Alterações realizadas:

```text
- Avatar do herói travado em formato quadrado.
- Desktop usando container estável de 96x96 px.
- Tablet usando container de 88x88 px.
- Mobile usando container de 80x80 px.
- Imagens de herói e boss passam a carregar como eager para evitar atraso visual em área visível.
- O fallback de imagem continua preservado para caso algum asset realmente falhe.
- Após renderizar os resultados, o sistema ativa uma verificação de imagens quebradas.
```

---

## 5. Grid dos cards dos heróis

O grid dos heróis foi ajustado para ser mais flexível.

Alterações realizadas:

```text
- Grid passou a usar auto-fit com largura mínima segura.
- Desktop continua permitindo até 4 cards quando há espaço.
- Telas menores quebram naturalmente para 2 ou 1 coluna.
- O card não deve mais comprimir o avatar.
```

---

## 6. Chains da composição

O bloco de chains foi refinado visualmente.

Alterações realizadas:

```text
- Chains agora ocupam uma largura mais confortável.
- P1, P2 e demais fases ganharam destaque visual.
- Cada chain foi organizada como item interno com rótulo e texto separado.
- Textos longos usam quebra controlada.
- Observações ficam em bloco separado quando existem.
```
