# Relatório de Atualização — Portal Avalon V7.0

## Integração Worker GTales.top e Refinamento da Aba Raid

## 1. Objetivo

A versão **Portal Avalon V7.0** iniciou oficialmente a etapa de integração estável da aba **Raid** com o Worker Cloudflare criado para intermediar as chamadas à GTales.top API.

Além da integração, a atualização refinou a exibição das composições para deixar a página mais simples, legível e coerente com os dados realmente fornecidos pela API.

---

## 2. Arquivos alterados

```text
web/assets/js/raid.js
web/assets/css/styles.css
READM_AVALON_VF_1.md
RELATORIO_V7_0_ATUALIZACAO.md
```

---

## 3. Integração com Worker

A aba Raid mantém o uso do Worker Cloudflare como base da API:

```text
RAID_API_BASE = Worker Cloudflare
RAID_ASSET_BASE = GTales.top assets
```

A URL de assets da GTales.top foi preservada para imagens de bosses, heróis, armas e acessórios.

As chaves de cache local foram atualizadas para V7.0, evitando que consultas antigas da V6 sejam reaproveitadas indevidamente.

---

## 4. Refinamento dos cards de heróis

Os cards de heróis foram compactados e receberam uma escala de imagem segura.

Antes, o card reservava um quadro retangular grande para a imagem do herói. Isso causava área vazia quando o asset não carregava ou quando a imagem não se comportava como banner.

Na V7.0:

```text
Desktop: container de herói 96x96 px
Mobile: container de herói 80x80 px
```

A imagem usa `object-fit: contain`, preservando o enquadramento do asset.

---

## 5. Fallback de imagens

Foi adicionada uma estratégia de fallback para tentar múltiplos caminhos possíveis de assets dos heróis.

Se nenhuma imagem carregar, o card exibe um placeholder pequeno e discreto, mantendo o nome do herói visível e impedindo que o layout fique com quadro grande vazio.

---

## 6. Dados exibidos corretamente

A API fornece dano por composição, não por personagem.

Por isso, a V7.0 remove dos cards individuais os campos:

```text
Dano individual
Fever individual
```

O cabeçalho da composição agora destaca:

```text
Dano total
Fever total
```

---

## 7. Chains e observações

As chains agora aparecem em um bloco próprio chamado **Chains da composição**.

As observações vindas do campo `infos` são exibidas somente quando existem. O HTML externo é convertido para texto limpo antes da exibição.

---

## 8. Fonte oficial GTales.top

Cada composição recebeu o botão:

```text
Ver no Guardian Tales TOP
```

O botão abre a página oficial da GTales.top para o boss e elemento consultados, ajudando a divulgar a fonte dos dados e permitindo conferência direta.

O botão de vídeo continua aparecendo somente quando a API retorna um link válido.
