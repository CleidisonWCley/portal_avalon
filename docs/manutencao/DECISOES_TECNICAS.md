# Decisões técnicas

1. **Aplicação estática:** o Portal permanece sem backend próprio para facilitar GitHub/Cloudflare Pages.
2. **Sem framework/build obrigatório:** HTML, CSS e JavaScript podem ser mantidos diretamente.
3. **JSON como fonte de dados publicada:** simplifica auditoria e backup.
4. **V7.6 funcional congelada:** V7.6.1 documenta e protege, não redesenha.
5. **Patentes dinâmicas:** pertencem à posição da raid, não ao jogador.
6. **Frequência oculta no Hall:** continua na lógica e aparece na Busca/Registro.
7. **Histórico preservado:** saída de membro não autoriza apagar evidência histórica.
8. **Componentes globais:** plataforma dos mascotes, feedback e botão de topo são compartilhados.
9. **Status inline da Raid separado:** `raid-status-card` e `raid-state-card` têm função diferente do feedback central.
10. **Cabeçalho e rodapé duplicados por página:** decisão aceitável para projeto estático sem template/build. Ao alterá-los, atualizar todas as páginas.
11. **GitHub como fonte e Cloudflare Pages como publicação recomendada:** o deploy deve apontar para `web/`.
12. **Documentação é fonte técnica:** conversas antigas não são pré-requisito para manutenção.
