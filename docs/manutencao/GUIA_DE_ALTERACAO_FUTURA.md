# Checklist de alteração futura

## Antes de alterar

- [ ] backup/branch;
- [ ] documentação relacionada lida;
- [ ] busca global por classes/funções;
- [ ] consumidores identificados;
- [ ] comportamento atual registrado.

## Durante

- [ ] componente existente reutilizado;
- [ ] regra de negócio separada do visual;
- [ ] histórico alterado somente com evidência;
- [ ] alteração pequena e reversível.

## Depois

- [ ] sintaxe e JSON;
- [ ] testes estruturais;
- [ ] desktop/mobile;
- [ ] canvas quando aplicável;
- [ ] CHANGELOG/release;
- [ ] evidências.

## Antes de publicar

- [ ] diff revisado;
- [ ] pasta limpa testada;
- [ ] ZIP validado;
- [ ] deploy conferido;
- [ ] rollback possível.
